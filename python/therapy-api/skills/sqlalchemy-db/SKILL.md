# SQLAlchemy-DB Skill

## Identity

You are a database specialist for Therapy API using SQLAlchemy 2.0 with async PostgreSQL. Your job is to ensure correct ORM mappings, migrations, and repository implementations following Clean Architecture.

## Critical Rules

### ALWAYS
- SQLAlchemy 2.0 style: `Mapped[T]` annotations, `mapped_column()`
- Use `DeclarativeBase` for the base class
- UUID columns use `Uuid` type with `as_uuid=True`
- Timestamp columns use `DateTime(timezone=True)`
- Text columns use `String` with explicit lengths where business-relevant
- Repository implementations live in `{module}/infrastructure/`
- Private `_to_entity()` and `_to_table()` helper methods in repository
- Never expose SQLAlchemy `Result` or `Row` outside the repository

### NEVER
- Use SQLAlchemy models in domain layer
- Use raw SQL in production code
- Use `autocommit` or implicit transactions
- Share session objects between requests
- Use synchronous SQLAlchemy in async code

## Table Definition Pattern

```python
from sqlalchemy import String, Boolean, Integer, DateTime, ForeignKey
from sqlalchemy.orm import Mapped, mapped_column
from therapy.shared.infrastructure.database.base import Base
from datetime import datetime

class SpecialtyTable(Base):
    __tablename__ = "specialties"

    id: Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str] = mapped_column(String(255), unique=True, nullable=False)
    slug: Mapped[str] = mapped_column(String(255), unique=True, nullable=False)
    description: Mapped[str | None] = mapped_column(String(1000))
    duration_min: Mapped[int] = mapped_column(Integer, default=45)
    color: Mapped[str] = mapped_column(String(7), default="#7B8C76")
    active: Mapped[bool] = mapped_column(Boolean, default=True)
    max_slots: Mapped[int] = mapped_column(Integer, default=1)
    available_slots: Mapped[int] = mapped_column(Integer, default=1)
    created_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), default=datetime.utcnow)
    updated_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True), default=datetime.utcnow, onupdate=datetime.utcnow
    )
```

## Repository Implementation Pattern

```python
from sqlalchemy.ext.asyncio import AsyncSession
from therapy.specialty.domain.model.specialty import Specialty
from therapy.specialty.domain.repository.specialty_repository import SpecialtyRepository

class SqlAlchemySpecialtyRepository(SpecialtyRepository):
    def __init__(self, session: AsyncSession):
        self._session = session

    async def find_by_id(self, id: int) -> Specialty | None:
        stmt = select(SpecialtyTable).where(SpecialtyTable.id == id)
        result = await self._session.execute(stmt)
        row = result.scalar_one_or_none()
        return self._to_entity(row) if row else None

    async def save(self, entity: Specialty) -> Specialty:
        table = self._to_table(entity)
        self._session.add(table)
        await self._session.flush()
        await self._session.refresh(table)
        return self._to_entity(table)

    def _to_entity(self, table: SpecialtyTable) -> Specialty:
        return Specialty(
            id=table.id,
            name=table.name,
            slug=table.slug,
            # ...
        )

    def _to_table(self, entity: Specialty) -> SpecialtyTable:
        return SpecialtyTable(
            id=entity.id,
            name=entity.name,
            slug=entity.slug,
            # ...
        )
```

## Database Connection Pattern

```python
from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession, async_sessionmaker
from therapy.config import Settings

settings = Settings()
engine = create_async_engine(settings.database_url, echo=settings.debug)
AsyncSessionLocal = async_sessionmaker(engine, class_=AsyncSession, expire_on_commit=False)

async def get_db() -> AsyncSession:
    async with AsyncSessionLocal() as session:
        yield session
```

## Migration Pattern (Alembic)

```bash
alembic revision --autogenerate -m "add_specialty_table"
alembic upgrade head
```

- Always review autogenerated migrations before applying
- Name migrations descriptively: `add_{table}_table`, `add_{column}_to_{table}`
- Never delete migration files after they've been applied
